`IEmbeddings` turns text into fixed-dimensional vectors for semantic search, clustering, and RAG pipelines. Every vector-store workflow in the platform starts with an embeddings adapter.

Source of truth: [platform/kb-labs-core/packages/core-platform/src/adapters/embeddings.ts](platform/kb-labs-core/packages/core-platform/src/adapters/embeddings.ts).

## Interface

```ts
interface IEmbeddings {
  embed(text: string): Promise<number[]>;
  embedBatch(texts: string[]): Promise<number[][]>;
  readonly dimensions: number;
  getDimensions(): Promise<number>;
}
```

Everything is required. The interface is deliberately tiny — embeddings adapters are thin wrappers over vendor APIs with no optional surface.

## Methods

### `embed(text)`

Generates a single embedding vector for the given text. Returns a `number[]` whose length equals `dimensions`.

```ts
const vector = await embeddings.embed('Hello, world!');
console.log(vector.length); // e.g., 1536 for OpenAI text-embedding-3-small
```

### `embedBatch(texts)`

Generates vectors for multiple texts in one call. Returns an array of vectors in the **same order** as the input. Batching is almost always cheaper than one-by-one — use this when you have more than a couple of texts.

```ts
const vectors = await embeddings.embedBatch([
  'first chunk',
  'second chunk',
  'third chunk',
]);
// vectors[0] corresponds to 'first chunk', etc.
```

Adapters that don't support batching natively should still implement this by calling `embed()` in a loop — the contract requires it.

### `dimensions` (property)

A synchronous, constant property giving the vector dimension. Used to size vector-store indexes and to validate inputs downstream. The value doesn't change after construction.

```ts
const indexSize = embeddings.dimensions; // 1536
```

### `getDimensions()`

The async equivalent of the `dimensions` property. This exists because property access doesn't survive IPC/Unix-socket transport — when an adapter runs in a subprocess and the caller talks to it over a socket, they need a method call to retrieve the value. Implementations should return the same number as the property.

```ts
const size = await embeddings.getDimensions(); // 1536
```

## Contract rules

- **Vectors must be stable for the same input.** Within a single model and model version, `embed('foo')` always returns the same vector. Caching layers depend on this.
- **`embedBatch` order is preserved.** `result[i]` corresponds to `texts[i]`. Adapters must not reorder.
- **`dimensions` is immutable.** A single adapter instance serves one model and one dimension count. If you switch models, create a new adapter instance.
- **Failure mode: throw.** There is no "partial result" — a batch call either returns every vector or throws.

## Writing an `IEmbeddings` adapter

```ts
import type { AdapterManifest, AdapterFactory, IEmbeddings } from '@kb-labs/core-platform';

export const manifest: AdapterManifest = {
  manifestVersion: '1.0.0',
  id: 'my-embeddings',
  name: 'My Embeddings',
  version: '0.1.0',
  type: 'core',
  implements: 'IEmbeddings',
  capabilities: { batch: true },
};

interface MyEmbeddingsConfig {
  apiKey: string;
  model: string;
  dimensions: number;
}

class MyEmbeddings implements IEmbeddings {
  readonly dimensions: number;

  constructor(private config: MyEmbeddingsConfig) {
    this.dimensions = config.dimensions;
  }

  async embed(text: string): Promise<number[]> {
    const [vec] = await this.embedBatch([text]);
    return vec;
  }

  async embedBatch(texts: string[]): Promise<number[][]> {
    const res = await fetch('https://api.my-embeddings.example.com/v1/embed', {
      method: 'POST',
      headers: {
        'Authorization': `Bearer ${this.config.apiKey}`,
        'Content-Type': 'application/json',
      },
      body: JSON.stringify({ model: this.config.model, input: texts }),
    });
    const data = await res.json();
    return data.data.map((d: { embedding: number[] }) => d.embedding);
  }

  async getDimensions(): Promise<number> {
    return this.dimensions;
  }
}

export const createAdapter: AdapterFactory<MyEmbeddingsConfig, {}, IEmbeddings> = (config) => {
  return new MyEmbeddings(config);
};
```
