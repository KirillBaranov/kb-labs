`IEventBus` is the platform's pub/sub layer. Services and plugins publish events to topics; other services and plugins subscribe with handlers. It's deliberately minimal — two methods — so adapters can map cleanly onto any pub/sub backend (in-memory, Redis, Kafka).

Source of truth: [platform/kb-labs-core/packages/core-platform/src/adapters/event-bus.ts](platform/kb-labs-core/packages/core-platform/src/adapters/event-bus.ts).

## Interface

```ts
type EventHandler<T> = (event: T) => Promise<void>;
type Unsubscribe = () => void;

interface IEventBus {
  publish<T>(topic: string, event: T): Promise<void>;
  subscribe<T>(topic: string, handler: EventHandler<T>): Unsubscribe;
}
```

That's the whole interface.

## Methods

### `publish<T>(topic, event)`

Broadcasts an event to all subscribers of the topic. Returns when the adapter has delivered (or queued) the event — delivery semantics depend on the backend:

- **MemoryEventBus** — awaits all subscribers synchronously (fan-out complete when the promise resolves).
- **Cache-backed bus** — returns once the event is written to the backing cache channel; subscribers pick it up asynchronously.
- **Future Redis/Kafka adapters** — will return once the event is acknowledged by the broker.

```ts
await eventBus.publish('workflow.completed', {
  runId: 'run-123',
  durationMs: 1234,
  status: 'success',
});
```

### `subscribe<T>(topic, handler)`

Registers an async handler for a topic. Returns an `Unsubscribe` function — call it to remove the handler.

```ts
const unsubscribe = eventBus.subscribe<WorkflowCompletedEvent>('workflow.completed', async (event) => {
  logger.info('workflow done', { runId: event.runId });
});

// Later
unsubscribe();
```

### Wildcards

Topic patterns support wildcards — `workflow.*` subscribes to all `workflow.<anything>` events. Adapters with native wildcard support (Redis pub/sub) use it directly; in-memory implementations regex-match at subscription time.

```ts
eventBus.subscribe('workflow.*', handler);        // all workflow events
eventBus.subscribe('workflow.*.failed', handler); // only failures
```

## Contract rules

- **Handlers are async.** Errors thrown by handlers are caught and logged by the bus — they don't propagate to the publisher.
- **`subscribe` returns a function, not a promise.** Unsubscription is synchronous.
- **Topics are strings.** No schema, no hierarchical enforcement — the `.` convention is just a convention.
- **Delivery guarantees depend on the adapter.** In-memory is at-most-once-per-subscriber-at-publish-time; cache-backed is best-effort; Redis/Kafka can provide at-least-once.
- **Event payloads are opaque.** The bus doesn't serialize or validate — plugins agree on schemas out-of-band.

## Writing an `IEventBus` adapter

A minimal in-memory implementation:

```ts
import type { AdapterManifest, AdapterFactory, IEventBus, EventHandler, Unsubscribe } from '@kb-labs/core-platform';

export const manifest: AdapterManifest = {
  manifestVersion: '1.0.0',
  id: 'memory-event-bus',
  name: 'In-Memory Event Bus',
  version: '0.1.0',
  type: 'core',
  implements: 'IEventBus',
  capabilities: {},
};

class MemoryEventBus implements IEventBus {
  private subs = new Map<string, Set<EventHandler<any>>>();

  async publish<T>(topic: string, event: T): Promise<void> {
    for (const [pattern, handlers] of this.subs) {
      if (this.matches(topic, pattern)) {
        await Promise.all(Array.from(handlers).map(h => h(event).catch(err => {
          // Log but don't propagate
          console.error(`Event handler failed for ${topic}:`, err);
        })));
      }
    }
  }

  subscribe<T>(topic: string, handler: EventHandler<T>): Unsubscribe {
    let set = this.subs.get(topic);
    if (!set) { set = new Set(); this.subs.set(topic, set); }
    set.add(handler);
    return () => set!.delete(handler);
  }

  private matches(topic: string, pattern: string): boolean {
    if (pattern === topic) return true;
    const regex = new RegExp('^' + pattern.replace(/\./g, '\\.').replace(/\*/g, '[^.]+') + '$');
    return regex.test(topic);
  }
}

export const createAdapter: AdapterFactory<unknown, {}, IEventBus> = () => new MemoryEventBus();
```
