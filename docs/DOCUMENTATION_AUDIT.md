## Common Update Patterns

### For README.md:
1. Add/update "Related Packages" section:
   ```markdown
   ## 🔗 Related Packages
   
   ### Dependencies
   - [@kb-labs/core](https://github.com/KirillBaranov/kb-labs-core) - Core utilities
   
   ### Used By
   - [Project](https://github.com/KirillBaranov/kb-labs-project) - Description
   
   ### Ecosystem
   - [KB Labs](https://github.com/KirillBaranov/kb-labs) - Main ecosystem repository
   ```

2. Update "Documentation" section:
   ```markdown
   ## 📚 Documentation
   
   - [Documentation Standard](./docs/DOCUMENTATION.md) - Full documentation guidelines
   - [Contributing Guide](./CONTRIBUTING.md) - How to contribute
   - [Architecture Decisions](./docs/adr/) - ADRs for this project
   ```

### For CONTRIBUTING.md:
1. Add reference to DOCUMENTATION.md in Architecture Decisions:
   ```markdown
   ### Architecture Decisions
   
   - For significant architectural changes, add an ADR in `docs/adr/`
   - Follow the ADR template in `docs/adr/0000-template.md`
   - Include required metadata (Date, Status, Deciders, Last Reviewed, Tags)
   - See [Documentation Standard](./docs/DOCUMENTATION.md) for ADR format requirements
   ```
