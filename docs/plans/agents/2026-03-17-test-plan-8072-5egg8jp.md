## Task

**From → To:** Execute and validate the full test suite across the KB Labs monorepo (`@kb-labs/workspace`), confirm all tests pass, identify any failures, and produce a coverage baseline — ready for ongoing CI.

The workspace consists of five main sub-repos under `platform/`:
- `kb-labs-core` — runtime, config, policy, contracts, platform adapters
- `kb-labs-cli` — CLI commands, bin, core, runtime, contracts, API
- `kb-labs-shared` — command-kit, CLI UI, testing utilities, utils
- `kb-labs-workflow` — engine, builtins, daemon, runtime, contracts, CLI
- `kb-labs-sdk`, `kb-labs-rest-api`, `kb-labs-studio` — SDK, REST API, Studio UI

All sub-repos use **Vitest** (`^3.2.4`) as the test runner, and the workspace is managed with **pnpm** (`^9.0.0`).

## Phase 1 — Verify Test Infrastructure

Before running tests, confirm the environment is healthy and all workspace packages are built — some tests import from `dist/` artifacts of sibling packages.

**Step 1.1 — Install dependencies**

`pnpm install`

This triggers `postinstall → pnpm devkit:sync` which synchronises the devkit tooling (tsconfig paths, vitest presets) across all sub-repos. The `|| true` guard in `package.json:31` means a sync failure won't block install in offline environments.

**Step 1.2 — Build all packages**

Tests in `core-runtime`, `cli-commands`, and `workflow-daemon` import compiled artifacts from sibling packages. The e2e tests — `platform/kb-labs-core/packages/core-runtime/src/__tests__/environment-init-platform.e2e.test.ts` and `platform/kb-labs-workflow/packages/workflow-daemon/src/__tests__/worker-lifecycle.e2e.test.ts` — especially require compiled output.

`pnpm build`

This runs `pnpm -r --filter ./packages/... run build` (root `package.json:38`) in topological order.

**Step 1.3 — Verify TypeScript compiles cleanly**

Catches type errors that would make tests pass for the wrong reasons. Each sub-repo runs `tsc -b tsconfig.tools.json`.

`pnpm type-check`

## Phase 2 — Run Platform Tests by Sub-repo

Running per sub-repo gives clearer failure isolation before doing the full sweep. Each sub-repo's `"test"` script is `vitest run --passWithNoTests`.

**Step 2.1 — kb-labs-core** (heaviest test suite)

`cd platform/kb-labs-core && pnpm test`

Covers: `core-runtime` (~10 test files including `container.test.ts`, `config-schemas.test.ts`, `service-bootstrap.test.ts`, `adapter-loader.test.ts`, `environment-lease-store.test.ts`), `core-config` (~10 spec files), `core-policy` (3 specs), `core-contracts` (3 tests), `core-platform` (3 analytics specs), `core-workspace`, `core-bundle`, `core-sys`, `llm-router`.

**Step 2.2 — kb-labs-shared** (command-kit, testing utilities)

`cd platform/kb-labs-shared && pnpm test`

Covers: `shared-command-kit` (`flags.spec.ts`, `errors.spec.ts`, `define-system-command.spec.ts`, `plugin-context-v3.snapshot.spec.ts`), `shared-tool-kit`, `shared-utils`, `shared-perm-presets`, `shared-testing`.

**Step 2.3 — kb-labs-workflow** (engine, builtins, daemon, runtime, contracts)

`cd platform/kb-labs-workflow && pnpm test`

Covers: `workflow-engine/src/__tests__/engine.test.ts`, `workflow-daemon` (host-service test, e2e lifecycle, `api-contract.integration.test.ts`), `workflow-builtins` (shell, gate, approval), `workflow-runtime` (workspace registry, output normalizer), `workflow-contracts` (expressions, constants).

**Step 2.4 — kb-labs-cli** (CLI commands, core, bin, runtime)

`cd platform/kb-labs-cli && pnpm test`

The `vitest.config.ts` here resolves `@kb-labs/plugin-adapter-cli` and `@kb-labs/plugin-adapter-rest` via aliases pointing into the `plugins/` sub-tree — these must be present (covered by `pnpm install` at root).

**Step 2.5 — Full workspace sweep**

`pnpm test`

Runs `pnpm -r --if-present run test` across every workspace package. A clean pass here is the primary go/no-go signal.

## Risks

| Risk | Impact | Mitigation |
|------|--------|------------|
| E2E tests in `workflow-daemon` (`worker-lifecycle.e2e.test.ts`) and `core-runtime` (`environment-init-platform.e2e.test.ts`) need a running Qdrant instance | Tests fail or hang | Run `pnpm dev:start:infra` first; or set `QDRANT_URL` env var to a remote instance |
| `api-contract.integration.test.ts` in `workflow-daemon` may require the REST API to be up | Integration test failures | Run `pnpm rest:dev` first in a separate terminal, or check if the test has an env-guard skip |
| `kb-labs-cli/vitest.config.ts:19–20` resolves plugin adapters via alias into `plugins/` sub-repo — must be installed | Module resolution failure at test start | `pnpm install` from workspace root handles this via `pnpm-workspace.yaml:7–8` |
| Build order: `core-contracts` must compile before `core-runtime` | Mid-build failure if run manually per-package | Always use `pnpm build` from root — topological order is handled automatically |
